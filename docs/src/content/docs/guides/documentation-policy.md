# ドキュメント方針

このドキュメントは、実装の前提と意思決定を後から追えるよう整備します。

## 書くもの

- プロダクト仕様
- 疑似CLIコマンド仕様
- Chrome Bookmarks APIとの連携仕様
- ユースケース
- ドメインモデル
- ブラウザ拡張としての権限と制約
- テスト方針

## 書き方

- 実装と同じ言葉を使う
- 未確定の仮説は、確定事項と分けて書く
- 仕様変更が起きたら、古い前提を残さず更新する
- 仕様ファイルはdocs配下に置き、Starlightのページとして読める状態にする
- 純粋関数にロジックを置く場合は、テスト観点も合わせて整理する
